# Product MVP Overview

## 1. Project Name
**EduFlow**

---

## 2. One-Sentence Pitch (Elevator Pitch)
EduFlow — образовательная платформа с AI-ассистентом, который подстраивает объяснения под стиль обучения, уровень и предпочтения пользователя на основе опросника и профиля.

*"An AI-powered educational assistant that personalizes explanations based on the user's learning style, level, and profile from an onboarding questionnaire."*

---

## 3. Problem Statement

- **Какова проблема сегодня?**  
  Студенты и самоучки сталкиваются с тем, что обычные AI-чаты дают один и тот же тип ответов: без учёта уровня, без выбора стиля (метафоры, примеры, краткость) и тона. Сложные темы объясняются либо слишком поверхностно, либо перегружено.

- **Почему это важно?**  
  Эффективность обучения сильно зависит от того, насколько объяснение совпадает с тем, как человек привык воспринимать информацию. Универсальный ответ подходит не всем.

- **Кто страдает?**  
  Школьники и студенты, готовящиеся к экзаменам; взрослые, осваивающие новые области; пользователи, которым нужны то короткие ответы, то подробные лекции — все, кому не хватает персонализации в обычных AI-чатах.

**Фокус:**
- Сложность понимания сложных тем без подстройки под пользователя
- Одинаковые AI-объяснения без адаптации
- Отсутствие персонализированного обучения в одном продукте

---

## 4. Our Solution

- **Идея:**  
  Один продукт: лендинг → регистрация → опросник персонализации → дашборд с курсами и AI-ассистентом. Ответы опросника сохраняются в профиле и передаются в системный промпт чата, чтобы модель меняла стиль, глубину, тон и использование метафор.

- **Ключевое отличие:**  
  Не просто чат с AI, а **чат, который знает пользователя**: уровень (новичок / средний / продвинутый), желаемый стиль (простые слова, примеры, метафоры, логика), глубину (коротко / средне / лекция), темп, тон, цели обучения и даже то, что раздражает (слишком сложно, много воды, мало примеров). Всё это влияет на каждый ответ.

- **Отличие от обычных AI-чатов:**  
  В обычных чатах промпт статичен. В EduFlow системный промпт собирается динамически из `preferences` в БД, поэтому один и тот же вопрос двух пользователей получает разные по стилю и глубине ответы.

---

## 5. Key Features (MVP Scope)

Только реализованный функционал:

- **Лендинг** — Hero, Features, How it works, Testimonials, Footer
- **Авторизация** — регистрация и вход через Supabase Auth
- **Пошаговый опросник онбординга** — один вопрос за раз, прогресс-бар, сохранение ответов в профиль (Supabase)
- **Профиль пользователя** — имя, фамилия, загрузка аватара (Supabase Storage), отображение имени везде в приложении
- **Контекст профиля (ProfileProvider)** — подгрузка профиля при входе в дашборд, имя и аватар в сайдбаре, приветствии, чате
- **Дашборд** — приветствие по имени, быстрые действия, дневные задачи (ссылки в AI-ассистент с предзаполненным запросом), прогресс по курсам, таймлайн активности
- **Курсы** — список курсов, страница курса с уроками, страница урока с квизами
- **Аналитика** — графики (Recharts): активность по дням, прогресс по месяцам, успеваемость по типам заданий
- **Достижения** — уровень, XP, стрики, список ачивок (геймификация)
- **AI-ассистент** — стриминг ответов (Vercel AI SDK + Groq), динамический системный промпт из опросника, обращение по имени, быстрые запросы (кнопка в инпуте), фиксированная нижняя панель, скролл только у сообщений
- **Интеграция с БД** — Supabase: таблица `profiles` (имя, фамилия, аватар, `preferences`), RLS, Storage для аватаров
- **Адаптивная вёрстка** — мобильный дашборд, нижняя навигация на малых экранах

## 6. How Personalization Works (CRITICAL SECTION)

- Пользователь проходит **онбординг-опросник**: уровень (новичок / средний / продвинутый), стиль объяснений (простые слова, примеры, метафоры, логика, структура), глубина (коротко / средне / подробно / лекция), нужны ли метафоры и какие (игры, техника, учёба, быт, наука, карьера), темп (медленно / средне / быстро), как объяснять сложные темы (упрощать / по шагам / сначала просто — потом глубже / сразу глубоко), цели (экзамены, школа, карьера, саморазвитие, быстрое понимание), тон (дружелюбный, наставник, академический, неформальный), язык (RU/EN), области обучения, раздражители (слишком сложно, много текста, мало примеров и т.д.), финальный стиль (как учителю новичку, с примерами из жизни, через метафоры, структурированно, кратко и ясно).

- Ответы сохраняются в **Supabase** в таблице `profiles`, поле `preferences` (JSONB).

- При каждом запросе в чат с клиента передаётся **текущий `preferences`** и **имя пользователя**. В **API** (`/api/chat`) из `preferences` собирается фрагмент системного промпта (`buildPersonaPrompt`): уровень, стиль, формат, метафоры, темп, сложные темы, цели, тон, язык и т.д. В системное сообщение добавляется также строка вида: «Пользователя зовут {имя}. Обращайся к нему по имени при необходимости.»

- В результате **каждый ответ модели** учитывает:
  - уровень пользователя,
  - выбранный стиль и глубину,
  - использование метафор и примеров,
  - темп и тон,
  - цели и раздражители.

Итог: **объяснения подстраиваются под конкретного пользователя**, а не под «среднего» слушателя.

## 7. Technical Architecture

**Frontend:**
- Next.js 16 (App Router), React 19, TypeScript
- Tailwind CSS, Radix UI, компоненты в стиле shadcn/ui
- Адаптивная вёрстка: сайдбар на десктопе, нижняя навигация на мобильных; фиксированная панель ввода в чате, скролл только у сообщений

**Backend:**
- API Route `/api/chat` — приём сообщений и `preferences`, сборка системного промпта, вызов Groq (OpenAI-совместимый API), стриминг ответа (Vercel AI SDK)
- API Route `/api/storage/ensure-avatars` — создание бакета Supabase Storage при наличии service role ключа

**AI:**
- Groq API (модель llama-3.3-70b-versatile), стриминг через `OpenAIStream` / `StreamingTextResponse`
- Динамический системный промпт: базовый текст + `buildPersonaPrompt(preferences)` + имя пользователя

**База данных:**
- Supabase (PostgreSQL): таблица `profiles` (id, first_name, last_name, avatar_url, questionnaire_completed_at, preferences, created_at, updated_at)
- RLS: пользователь видит и обновляет только свой профиль
- Триггеры: обновление `updated_at`, создание записи в `profiles` при регистрации

**Storage:**
- Supabase Storage, бакет `avatars` (публичный): загрузка аватара по пути `{user_id}/avatar.{ext}`, RLS-политики на чтение/запись/обновление/удаление по папке пользователя

**Авторизация:**
- Supabase Auth (email/password), интеграция с Redux (authApi) и контекстом профиля

## 8. AI Integration

- **Стриминг:** ответы приходят потоком (SSE), отображаются в реальном времени в чате (Vercel AI SDK `useChat`).

- **Динамический системный промпт:** при каждом запросе в тело передаются `preferences` (из профиля) и `userName`. На сервере вызывается `buildPersonaPrompt(preferences)` — формируется блок «Персонализация под пользователя» с уровнем, стилем, форматом, метафорами, темпом, сложными темами, целями, тоном, языком, областями, раздражителями и финальным стилем. Этот блок добавляется к базовому системному промпту; отдельно добавляется обращение по имени.

- **Контекст пользователя:** модель получает не только историю сообщений, но и явное описание того, как именно объяснять этому пользователю. В результате один и тот же вопрос от двух пользователей с разными `preferences` даёт разные по стилю и глубине ответы.

- **Итог:** AI не просто «используется» — он **адаптирует объяснения** под стиль обучения, уровень и предпочтения пользователя на основе данных опросника и профиля.

## 9. User Flow (Demo Scenario)

1. Пользователь заходит на лендинг, переходит к регистрации.
2. Регистрируется (email, пароль, имя, фамилия).
3. После входа может пройти опросник персонализации (по одному вопросу, с прогресс-баром) или пропустить.
4. Ответы сохраняются в профиль в Supabase.
5. В профиле пользователь указывает имя/фамилию и загружает аватар (файл → Supabase Storage).
6. Имя и аватар отображаются в сайдбаре, на дашборде («Welcome back, {имя}»), в чате с AI.
7. Пользователь открывает AI-ассистента: поле ввода закреплено внизу, быстрые запросы — кнопка в инпуте слева. Задаёт вопрос — получает ответ, подстроенный под свой уровень и стиль из опросника; модель обращается по имени.
8. Дашборд: дневные задачи ведут в ассистент с предзаполненным запросом; прогресс по курсам, аналитика, достижения — всё с учётом единого профиля и персонализации.

## 10. UX & Design Decisions

- **Единый дашборд:** сайдбар с именем и аватаром, навигация по разделам (Dashboard, Courses, Analytics, Achievements, AI Assistant, Profile).
- **Чат:** скролл только у области сообщений; нижняя панель (инпут + кнопка отправки + быстрые запросы) закреплена внизу — не уезжает при прокрутке.
- **Быстрые запросы:** одна кнопка (молния) в поле ввода слева от текста — быстрый доступ к типовым запросам без отдельного ряда кнопок.
- **Опросник:** один вопрос на экран, прогресс-бар сверху, кнопки «Назад» / «Далее» / «Сохранить и перейти» — снижение когнитивной нагрузки.
- **Профиль:** загрузка аватара через модалку с превью и сохранением в Storage; после сохранения контекст профиля обновляется — имя и аватар сразу видны в сайдбаре и чате.
- **Мобильная вёрстка:** дашборд и страницы с overflow-auto; на малых экранах нижняя навигация вместо сайдбара.

---

## 11. What Makes This Product Unique

- **Обычный AI-чат:** один системный промпт для всех; стиль и глубина не зависят от пользователя.

- **EduFlow:** системный промпт **собирается под пользователя** из опросника и профиля. Один и тот же вопрос «Объясни квантовую запутанность» для новичка с любовью к метафорам и для продвинутого пользователя с запросом «коротко и по делу» даёт разные по длине, тону и способу объяснения ответы.

- **Статичное обучение:** курсы и статьи не подстраиваются под уровень и стиль.

- **EduFlow:** AI-ассистент выступает как **адаптивный слой**: те же темы объясняются по-разному в зависимости от профиля.

- **Итог:** продукт выделяется формулой **«AI, который объясняет по-разному каждому пользователю»** за счёт явного профиля обучения и динамического промпта.

## 12. Challenges We Solved During Development

- **Стриминг и лимиты:** обработка ошибок Groq (в т.ч. 429), сообщение пользователю о необходимости подождать при исчерпании лимита.
- **Фиксированная панель чата:** разделение скролла (только сообщения) и фиксированного низа (инпут) через абсолютное позиционирование панели и отступ снизу у контента; отключение скролла у layout main для страницы ассистента.
- **Контекст профиля:** единая подгрузка профиля при входе в дашборд (ProfileProvider), обновление сайдбара, приветствия и чата без повторных запросов; вызов `refresh` после сохранения профиля и загрузки аватара.
- **Supabase Storage:** создание бакета через API (service role), RLS-политики для папки пользователя (`auth.jwt()->>'sub'`), загрузка аватара файлом с превью и сохранением URL в `profiles`.
- **Персонализация в промпте:** типобезопасная сборка блока персонализации из множества полей опросника (`buildPersonaPrompt`) и передача `preferences` с клиента в API при каждом запросе.

## 13. Future Improvements (Post-MVP)

- Расширенная аналитика обучения (время по темам, прогресс по целям из опросника).
- Персонализация на нескольких языках (сохранение языка в профиле и переключение интерфейса).
- Голосовой ввод/вывод в AI-ассистенте.
- Генерация персональных траекторий и рекомендаций курсов на основе профиля и прогресса.
- Кабинет преподавателя: просмотр агрегированных предпочтений учеников и эффективности форматов объяснений.

## 14. Target Audience

- **Студенты** — подготовка к экзаменам, разбор сложных тем с подстройкой под уровень и стиль.
- **Самоучки** — освоение новых областей с контролем глубины и темпа объяснений.
- **Образовательные платформы** — встраивание персонализированного AI-тьютора поверх своих курсов.
- **Новички в сложных темах** — пользователи, которым важны метафоры, примеры и пошаговость, а не один «академический» стиль для всех.

---

## 15. Conclusion

EduFlow — это не прототип идеи, а **работающий MVP**: регистрация, профиль с аватаром, опросник персонализации, дашборд с курсами и аналитикой, AI-ассистент со стримингом и **реальной адаптацией ответов** под уровень, стиль и предпочтения пользователя. Ключевая ценность — **персонализация образования через AI**: один интерфейс, разное объяснение для каждого. Стек (Next.js, Supabase, Groq, динамические промпты) позволяет масштабировать продукт и добавлять новые форматы обучения и аналитики поверх уже собранного профиля пользователя.
